Package Development Utilities

Usage

packageEnv(pkg)

packageName(.Global = FALSE)

packagePath(..., PACKAGE = NULL)

isPackageInstalled(..., lib.loc = NULL)

Arguments

pkg
package name. If missing the environment of the caller package is returned.
.Global
a logical that indicates if calls from the global environment should throw an error (FALSE: default) or the string 'R_GlobalEnv'.
PACKAGE
optional name of an installed package
...
arguments passed to file.path.
lib.loc
path to a library of R packages where to search the package

Value

an environment

a character string

a character string

Description

packageEnv is a slight modification from topenv, which returns the caller's top environment, which in the case of development packages is the environment into which the source files are loaded by load_all.

packageName returns the current package's name.

packagePath returns the current package's root directory, which is its installation/loading directory in the case of an installed package, or its source directory served by devtools.
